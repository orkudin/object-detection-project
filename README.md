# 🦅 UAV-CV-Navigator

Система обнаружения, отслеживания объектов и адаптивного планирования маршрута БПЛА для мониторинга воздушного пространства и взлётно-посадочных полос (ВПП).

![Демонстрация работы](docs/demo.gif)

Проект разработан в рамках научной стажировки МУИТ АГА (2026 г.) с упором на отказоустойчивость, модульность и энергоэффективность.

## 🚀 Быстрый старт (Для операторов)

Запустить проект можно всего в 3 простых шага. Веса нейросети загрузятся автоматически, вручную ничего скачивать не нужно! Вы можете просто скопировать эти команды в ваш терминал.

**Шаг 1. Скачайте проект и перейдите в папку**
```bash
git clone <repo_url>
cd object_detection
```

**Шаг 2. Создайте изолированное окружение** (Требуется Anaconda/Miniconda)
```bash
conda create -n uav_cv python=3.11 -y
conda activate uav_cv
pip install -r requirements.txt
```

**Шаг 3. Запустите интерфейс оператора**
```bash
python src/app.py
```
*После запуска откройте браузер и перейдите по ссылке: [http://127.0.0.1:7860](http://127.0.0.1:7860)*

Система обнаружения, отслеживания объектов и адаптивного планирования маршрута БПЛА для мониторинга воздушного пространства и взлётно-посадочных полос.

| Параметр | Значение |
|---|---|
| Статус | Прототип (ТЗ от 13.04.2026) |
| Целевая ОС | Ubuntu 20/22/24 LTS, Windows 10/11 |
| Python | 3.10+ (рекомендуется 3.11) |
| Платформы инференса | PC (CPU / GPU), Radxa Rock 4D (Rockchip RK3576 NPU) |

Архитектура проекта следует принципам SOLID и DRY. Все параметры вынесены в YAML-конфигурацию. Бизнес-логика декомпозирована на независимые модули: детекция, трекинг, планирование маршрута, визуализация.

## Ограничения системы (Scientific Constraints)
- **Условия среды:** Эффективность базовых весов детектора снижается при плохой освещенности (ночь, плотный туман, засветка). Для всепогодного применения требуется внедрение инфракрасных/тепловизионных сенсоров и дообучение модели.
- **Дальность обнаружения (Small Objects):** Стандартная модель `yolo11s` работает с изображениями (640x640), из-за чего удаленные объекты теряются в пиксельной сетке. Необходима поддержка Slicing Aided Hyper Inference (SAHI).
- **Отсутствие IMU:** Алгоритм планировщика маршрутов (Heatmap + Age Map) сейчас работает в системе координат кадра. Без глобальных координат GPS/IMU или алгоритмов стабилизации треков агрессивные маневры дрона приводят к смещению эвристической карты.

## 🖥️ Веб-интерфейс (Панель оператора)

Помимо консольного режима, система снабжена визуальным дашбордом на базе `Gradio` (запуск через `python src/app.py`).

**Ключевые возможности:**
- **Различные источники** — загрузка локального видео, просмотр RTSP-трансляции, подключение USB-камеры.
- **Гибкая настройка** — переключение конфигураций трекера (ByteTrack / BoT-SORT), порога уверенности и шага сетки навигатора в реальном времени.
- **Скрытие OSD** — возможность отключать инфопанель и прицел галочкой прямо из веб-интерфейса для чистого обзора ВПП.

## 💻 Продвинутый запуск (CLI)

Для серверов без графического интерфейса предусмотрен классический запуск скрипта из командной строки.

```bash
# Обязательно укажите PYTHONPATH (если не используете IDE)
export PYTHONPATH=$PYTHONPATH:$(pwd)

# Windows PowerShell
$env:PYTHONPATH += ";$(Get-Location)"
```

### Основной запуск

Благодаря переносу параметров в `configs/default.yaml` (Single Source of Truth), запуск осуществляется предельно лаконично, без длинных цепочек флагов:

```bash
# Применение переменных среды (опционально)
# source .env  # (Linux/macOS)

python src/pipeline.py 
```
*(Скрипт автоматически "подтянет" видео `data/test_airport.mp4` и необходимые режимы трекинга)*

**Переопределение параметров "на лету" (Override):**
Если вы хотите временно запустить другой сценарий без редактирования YAML-файла:
```bash
python src/pipeline.py --video data/my_drone_footage.mp4 --weights models/airplane_yolo.pt 
```

### Доступные флаги для оптимизации:
| Флаг | Описание |
|---|---|
| `--headless` | Отключает вывод окон OpenCV. **Обязательно** для серверов (Colab/VPS). |
| `--no-osd` | Скрывает отрисовку графической инфопанели (FPS, Режим) на видео, ускоряя рендер кадра. |
| `--no-target-wp` | Скрывает красный прицел (Target WP) навигатора. |

*Для остановки терминального сеанса в визуальном режиме нажмите клавишу `q`.*

## 🧩 Модульная архитектура (SOLID)

```text
                         pipeline.py
    ┌────────────┐    ┌────────────┐    ┌──────────────┐
    │  Detector  │ -> │  Tracker   │ -> │   Planner    │
    │  (Factory) │    │ (ByteTrack │    │  (Adaptive   │
    └──────┬─────┘    │  / BoTSORT)│    │   Heatmap)   │
           │          └────────────┘    └──────────────┘
    ┌──────┴──────────────────────┐
    │       BaseDetector (ABC)    │  <- Dependency Inversion
    ├─────────────┬───────────────┤
    │ YOLODetector│ RKNNDetector  │
    │ (.pt, .onnx)│ (.rknn, NPU)  │
    └─────────────┴───────────────┘
```

**Принцип работы фабрики (`src/detection/factory.py`):**
Скрипт автономно проверяет запрошенные веса. Если это `.pt` — поднимается `YOLODetector` (PyTorch). Если `.rknn` — `RKNNDetector` (адаптер для запуска на NPU-процессорах). Логика конвейера остается неизменной, так как все детекторы отдают унифицированный словарь данных через абстракцию `BaseDetector`.

## ⚙️ Архитектурная оптимизация и производительность

Архитектура проекта следует принципам SOLID и DRY. Конвейер спроектирован для работы на энергоэффективных платах, в нем внедрено мощное программно-аппаратное снижение вычислительной нагрузки:

1. **Geometry ROI (Регион Интереса ВПП):**
   Вместо того чтобы тяжелый детектор и трекер (`ByteTrack` / `BoT-SORT`) отслеживали мельчайший мусор по всему огромному кадру, мы внедрили математическую маску зоны посадки `runway_roi`. Объекты вне этой задаваемой трапеции геометрически отбрасываются *строго до входа в конвейер трекинга*, экономя от 30% до 50% CPU ресурсов.
2. **Динамическая фильтрация классов (Zero-Cost Inference):**
   Если в файле `configs/default.yaml` параметр `classes_of_interest` установлен в `null`, система автоматически парсит внутренний словарь загруженной модели нейросети и выводит красивые названия найденных сущностей (например, `AIRPLANE`, `PERSON`). Если переданы ID конкретных классов, система не тратит кэш-память на посторонние объекты.
3. **Адаптивный планировщик (`Adaptive Heatmap`):**
   Маршрут полета дрона-наблюдателя по территории аэропорта строится динамически на основе матрицы приоритетов: `P = W_det * D + W_time * T` (учитывается плотность детекции объектов и давность последнего инспекционного визита зоны).
4. **Единый источник истины (Single Source of Truth YAML):**
   Все системные константы, параметры локальной сети (`server_port`) и дефолтные значения графического интерфейса жестко отвязаны от Python кода. Всё управляется исключительно из файла `configs/default.yaml`, что позволяет масштабировать проект без вмешательства в программирование: "Пишем код один раз — конфигурируем всю жизнь".

## 🧠 Поддерживаемые модели

Система интегрирована с библиотекой Ultralytics. Вам не нужно искать или конвертировать файлы весов — всё происходит «под капотом».

| Модель | Формат | Описание |
|---|---|---|
| `yolo11n.pt` | PyTorch | Самая быстрая (Nano) конфигурация для Edge-устройств. Скачивается автоматически. |
| `yolo11s.pt` | PyTorch | Оптимальный (Small) баланс точности и FPS. Скачивается автоматически. |

> Любые тонкие параметры работы этих моделей, включая порог утверждения объектов (`confidence_threshold`) или метрики дистанции (`iou_threshold`), вынесены в единый файл конфигурации `configs/default.yaml`. Мультимодульная структура не требует изменения исходного Python-кода!
