# 🦅 UAV-CV-Navigator: Intelligent Aerial Object Detection & Tracking

Система обнаружения, отслеживания объектов и адаптивного планирования маршрута БПЛА. Разработано специально для мониторинга воздушного пространства и взлетно-посадочных полос (ВПП).

**Статус:** Прототип (Завершено Техническое Задание от 13.04.2026).  
**Целевая ОС:** Ubuntu 20.04/22.04/24.04 LTS (Протестировано в Cloud/Colab и локально).  
**Совместимость:** Python 3.10+ (рекомендуется 3.11).

> Разработано в строгом соответствии с концепцией **Zero Hardcode Policy**. В архитектуре разграничены логические узлы конвейера: **Детекция ≠ Трекинг ≠ Планирование**.

---

## 🚀 1. Установка и настройка окружения

Проект рекомендуется запускать в изолированной средe `conda`, что страхует от конфликтов транзитивных зависимостей (PyTorch, OpenCV, lapx).

```bash
conda create -n object_detection_project python=3.11 -y
conda activate object_detection_project
pip install -r requirements.txt
```

*(Опционально)* Скрипт для скачивания демо-видео БПЛА, если у вас нет своих записей:
```bash
python data/download_test_video.py
```

---

## 🛠 2. Конфигурирование (Zero Hardcode)

Вся бизнес-логика пайплайна конфигурируется исключительно через файл `configs/default.yaml`. Код в `src/*.py` менять не нужно!

**Секции конфига:**
- **`detector`**: Путь к весам (`mnv4_yolo11s.pt` и др.), настройки Confidence, а также фильтр `classes_of_interest` для исключения ложных срабатываний на мусорном фоне.
- **`tracker`**: Выбор алгоритма отслеживания (нативно поддерживаются `bytetrack` и `botsort`). Полный контроль над пороговыми значениями совпадений IoU/Motion.
- **`planner`**: Установка режима: `static` (поиск сеткой) или `adaptive` (эвристическое наведение по горячим точкам $P_{ij}$). Установка шага сетки и безопасной высоты.

## 🖥 3. Локальный Запуск

Точка входа — скрипт `src/pipeline.py`. 
*Важно: Зарегистрируйте корневую папку в памяти Python перед запуском.*

**Linux/Ubuntu:**
```bash
export PYTHONPATH=$PYTHONPATH:$(pwd)
```
**Windows (PowerShell/CMD):**
```cmd
set PYTHONPATH=%PYTHONPATH%;%cd%
```

### Запуск по ТЗ (Одной Командой)
```bash
python src/pipeline.py --video data/test_runway.mp4 --config configs/default.yaml --mode full
```

### Дополнительные режимы сценария:
- **Выбор собственной модели на лету:** Передайте путь к вашим `.pt` весам без редактирования YAML: `python src/pipeline.py --weights mnv4_yolo11s.pt`
- **Веб-камера / USB кастера:** Передайте `0` вместо пути: `python src/pipeline.py --video 0`
- **RTSP Поток с дрона:** `python src/pipeline.py --video "rtsp://192.168.1.100:554/stream"`
- **Переназначение трекера на лету:** `--tracker botsort` (отлично компенсирует тряску камеры встроенным оптическим потоком GMC).

> Во время Live-отрисовки нажмите `q` для грациозной остановки системы.

## ☁️ 4. Облачный Запуск (Google Colab демо)

Система изначально спроектирована "Headless-Safe" — она не упадет с `OpenCV: The function is not implemented`, если запустить ее на сервере без дисплея.

В директории `notebooks/` расположен файл **`Colab_Pipeline_Runner.ipynb`**.
Загрузите этот файл в Google Colab. Он содержит автоматические ячейки, которые:
1. Склонируют ваш репозиторий.
2. Подтянут системные зависимости `Ubuntu 22.04` (`libgl1-mesa-glx`).
3. Запустят пайплайн.
4. Выведут встроенным плеером итоговый видеофайл OSD.
5. Построят научные графики метрик.

## 🧪 5. Юнит-Тестирование системы

Архитектура покрыта `unittest` тестами, проверяющими устойчивость конвейера к сбоям на краевых случаях.

**Запуск тестов:**
```bash
python -m unittest discover tests
```

**Что покрывают тесты:**
- `test_config_validation`: Строгая валидация структуры `yaml` конфигов на предмет хардкода.
- `test_planner_route`: Изолированная отработка математической логики генерации динамических Waypoints в `adaptive` режиме на основе искусственно сгенерированной матрицы скоплений BBox.
- `test_tracking_stability`: Стресс-тест детектора на "битые", черные кадры и отсутствие объектов (патч бага Ultralytics `tensor(-14)`).

## 📊 6. Вывод и Телеметрия

В результате работы в папке `data/output/` появляются артефакты:
