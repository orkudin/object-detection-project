# UAV-CV-Navigator

Система обнаружения, отслеживания объектов и адаптивного планирования маршрута БПЛА для мониторинга воздушного пространства и взлётно-посадочных полос.

| Параметр | Значение |
|---|---|
| Статус | Прототип (ТЗ от 13.04.2026) |
| Целевая ОС | Ubuntu 20/22/24 LTS, Windows 10/11 |
| Python | 3.10+ (рекомендуется 3.11) |
| Платформы инференса | PC (CPU / GPU), Radxa Rock 4D (Rockchip RK3576 NPU) |

Архитектура проекта следует принципам SOLID и DRY. Все параметры вынесены в YAML-конфигурацию. Бизнес-логика декомпозирована на независимые модули: детекция, трекинг, планирование маршрута, визуализация.

## Конфигурирование

Все параметры пайплайна определяются в файле `configs/default.yaml`. Непосредственное редактирование исходного кода для изменения параметров не требуется.

| Секция | Ключевые параметры | Назначение |
|---|---|---|
| `detector` | `weights_path`, `confidence_threshold`, `classes_of_interest` | Путь к весам модели, порог уверенности, фильтр классов COCO |
| `tracker` | `algorithm`, `track_buffer`, `match_thresh` | Алгоритм трекинга (bytetrack / botsort), буфер памяти, порог IoU |
| `planner` | `mode`, `grid_step_meters`, `priority_weights` | Режим навигации (static / adaptive), шаг сетки, веса эвристики |
| `pipeline` | `telemetry_format`, `log_level` | Формат вывода телеметрии, уровень логирования |

## Запуск из терминала (CLI)

### Подготовка окружения

Необходимо однократно добавить корневую директорию проекта в `PYTHONPATH`:

```bash
# Linux / macOS / Colab
export PYTHONPATH=$PYTHONPATH:$(pwd)

# Windows PowerShell
$env:PYTHONPATH += ";$(Get-Location)"
```

### Основной запуск

```bash
python src/pipeline.py --video data/test_video.mp4 --config configs/default.yaml --mode full
```

### Параметры командной строки

| Флаг | Значение по умолчанию | Описание |
|---|---|---|
| `--video` | `data/test_video_2.mp4` | Путь к видеофайлу, индекс USB-камеры (`0`) или RTSP-ссылка |
| `--config` | `configs/vision_pipeline.yaml` | Путь к YAML-конфигурации |
| `--output_video` | `data/output/result.mp4` | Путь для сохранения результата |
| `--weights` | из конфига | Переопределяет `weights_path` из конфигурации |
| `--tracker` | из конфига | Переопределяет алгоритм трекера (`bytetrack` / `botsort`) |
| `--headless` | `false` | Отключает вывод окон (обязательно для серверов и Colab) |

Для остановки обработки в режиме реального времени нажмите клавишу `q`.

## Веб-интерфейс (Панель оператора)

Запуск:

```bash
python src/app.py
```

Интерфейс доступен по адресу `http://127.0.0.1:7860`.

Возможности панели оператора:

- **Источники видео** — загрузка локального файла, ввод RTSP-ссылки, подключение USB-камеры.
- **Выбор модели** — автоматическое сканирование папки `models/` с отображением доступных весов (`.pt`, `.onnx`, `.rknn`).
- **Настройка трекера** — переключение между ByteTrack и BoT-SORT.
- **Параметры детекции** — регулировка порога уверенности (Confidence Threshold).
- **Параметры навигации** — регулировка шага сетки маршрутизатора.
- **Управление** — запуск и остановка обработки.
- **Визуализация** — потоковое отображение обработанных кадров с OSD-телеметрией.

## Архитектура

```
                         pipeline.py
    ┌────────────┐    ┌────────────┐    ┌──────────────┐
    │  Detector  │ -> │  Tracker   │ -> │   Planner    │
    │  (Factory) │    │ (ByteTrack │    │  (Adaptive   │
    └──────┬─────┘    │  / BoTSORT)│    │   Heatmap)   │
           │          └────────────┘    └──────────────┘
    ┌──────┴──────────────────────┐
    │       BaseDetector (ABC)    │  <- Dependency Inversion
    ├─────────────┬───────────────┤
    │ YOLODetector│ RKNNDetector  │
    │ (.pt, .onnx)│ (.rknn, NPU)  │
    └─────────────┴───────────────┘
```

**Принцип работы фабрики** (`src/detection/factory.py`):

- Расширение `.pt` или `.onnx` — создаётся `YOLODetector` (инференс через PyTorch или ONNX Runtime).
- Расширение `.rknn` — создаётся `RKNNDetector` (инференс через rknnlite на NPU Rockchip).

Все детекторы реализуют единый контракт `BaseDetector.track_frame()`, возвращая стандартизированный массив `[{"track_id": int, "class_id": int, "bbox": [x, y, w, h]}]`.  Пайплайн не зависит от конкретной реализации детектора.

**Адаптивный маршрутизатор** (`src/planning/planner.py`):

- Режим `static` — обход территории по паттерну «газонокосилка» (lawnmower).
- Режим `adaptive` — генерация следующей точки маршрута на основе приоритетной матрицы:  
  `P(i,j) = W_det * D(i,j) + W_time * T(i,j)`, где `D` — тепловая карта детекций, `T` — матрица времени с последнего посещения.

## Модели

Файлы весов размещаются в папке `models/`. Веб-интерфейс автоматически обнаруживает все поддерживаемые файлы при запуске.

| Модель | Формат | Назначение |
|---|---|---|
| `yolo11s.pt` | PyTorch | Базовая детекция (COCO, 80 классов) |
| `yolo11s.onnx` | ONNX | Кроссплатформенный инференс с оптимизацией для CPU |
| `mnv4_yolo11s.pt` | PyTorch | Дообученная модель для аэросъёмки (MobileNetV4) |
| `mnv4_yolo11s.onnx` | ONNX | Экспорт MNv4 для развёртывания на Edge/Server |
| `*.rknn` | RKNN | Аппаратный инференс на NPU Rockchip RK3576 |

Стандартные веса `yolo11s.pt` загружаются автоматически при первом запуске. Веса не включаются в репозиторий (см. `.gitignore`).
